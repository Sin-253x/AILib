# 代码解释 + 注释模板（请严格遵守）

在生成或修改代码时，请同时完成以下两件事：

**A. 在对话中用中文回答（结构必须如下）：**
1. **整体功能**：用一句话总结这段代码主要做了什么。
2. **关键部分拆解**：列出主要函数、组件、类或逻辑块，并说明每个部分的作用。
3. **重要概念与库**：列出用到的核心库和概念，并用简单语言解释它们是什么、为什么在这里使用。
4. **潜在问题与改进建议**：指出代码中可能存在的 bug、性能问题、安全隐患、代码可读性问题，并给出改进方向。
5. **修改指南**：如果我想自己修改或扩展这个功能，应该从哪里入手？请给出具体的修改步骤或建议。

**B. 在代码文件中以注释形式写入解释（必须执行）：**

请在生成的代码中，以**结构化注释**的形式，把以上 5 点解释内容写入代码文件。

**注释格式要求：**
- **Python 文件**：使用多行 docstring `"""` 或 `#` 注释
- **TypeScript / JavaScript / JSX 文件**：使用 JSDoc `/** */` 或 `//` 注释
- 注释要放在**函数、组件、类或关键逻辑块的正上方**
- 注释内容要**简洁但完整**，不要过度冗长影响代码阅读
- 结构必须保持以下格式：

```python
# ======================== 代码解释 ========================
# 1. 整体功能：
#    [一句话总结]
#
# 2. 关键部分拆解：
#    - xxx：作用是...
#    - yyy：作用是...
#
# 3. 重要概念与库：
#    - LangChain：用于构建 AI 应用链...
#    - pgvector：PostgreSQL 的向量扩展...
#
# 4. 潜在问题与改进建议：
#    - ...
#
# 5. 修改指南：
#    - 如果想修改...，建议从...入手
# ========================================================
